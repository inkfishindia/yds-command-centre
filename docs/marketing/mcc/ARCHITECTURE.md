## Key Components

### 1. Social Provider Layer (`server/services/social/`)

Pure modules implementing the provider interface:
- `instagram.js` вҖ” Instagram Graph API (stub in P1)
- `linkedin.js` вҖ” LinkedIn Marketing API (stub in P1)
- `validate-media.js` вҖ” Media constraint validation
- `media-constraints.json` вҖ” Platform-specific rules

**Interface Contract:**
```javascript
{
  getProviderInfo() вҶ’ { id, name },
  login(redirectUri) вҶ’ { authUrl, state },
  refreshToken(refreshToken) вҶ’ { accessToken, expiresAt },
  post(accessToken, payload) вҶ’ { platformPostId, url },
  getAnalytics(accessToken, postId) вҶ’ { likes, comments, shares, views }
}
```

### 2. MCC Domain Service (`server/services/mcc/`)

- `read-model.js` вҖ” Lists posts by status, filters
- `drafter.js` вҖ” Create/update/delete drafts
- `scheduler.js` вҖ” Schedule/unschedule with validation
- `publisher.js` вҖ” Publish workflow with approval

### 3. Routes (`server/routes/mcc.js`)

Thin route layer with:
- SSE-based approval gate (`runWithApproval`)
- CRUD operations
- OAuth start/callback

### 4. Frontend (`src/js/modules/mcc.js`)

Alpine.js module with:
- Post list by status (kanban)
- Composer modal with preview
- Platform selection
- OAuth connection status

## State Machine

```
draft вҶ’ scheduled вҶ’ awaiting-approval вҶ’ publishing вҶ’ published
                                          вҶҳ failed
```

## Approval Gate Flow

1. **Request** вҖ” User triggers action (create, schedule, publish)
2. **SSE Response** вҖ” Server emits `approval` event
3. **UI Shows** вҖ” Dan sees approval prompt with summary
4. **Decision** вҖ” Dan approves/rejects via `/api/chat/approve`
5. **Execute** вҖ” On approval, operation runs вҶ’ Notion updated

## Patterns Used

- **Thin routes** вҖ” HTTP validation only, business logic in services
- **Pure modules** вҖ” No classes, no inheritance in social providers
- **Read-model** вҖ” Lists use read-model pattern, single fetches use direct Notion
- **SSE streaming** вҖ” Approval gate uses Server-Sent Events
- **Lazy loading** вҖ” Module and CSS loaded on first view access
