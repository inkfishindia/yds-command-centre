## Strict Principles

1. **No hardcoded colors** — Use CSS variables (`--accent`, `--teal`, …) from `TOKENS.md`
2. **No emoji icons** — SVG only (Lucide / Heroicons). Section emoji in copy is OK.
3. **No new fonts** — `Inter` (`--font-ui`) + `IBM Plex Mono` (`--font-mono`)
4. **No frameworks** — Vanilla CSS, BEM naming, Alpine.js
5. **No new tokens without justification** — Extend the palette only when an existing token can't carry the meaning
6. **Accessibility mandatory** — 4.5:1 contrast, 44px touch targets, visible focus rings, `prefers-reduced-motion`
7. **Approval gate required** — All write paths surface the amber approval banner
8. **8px baseline** — Spacing in multiples of 4px or 8px

## Workflow

`design-planner` writes the brief here → `frontend-builder` implements against `TOKENS.md` + `CSS-PATTERNS.md` → `code-reviewer` + `ux-auditor` verify against the brief.
